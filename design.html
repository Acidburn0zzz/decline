  • Functional: decline aims to be a good functional-programming citizen: interfaces should be immutable, and typeclass instances should be provided and lawful.

  • GNU/POSIX: As much as possible, applications built with decline should behave like any other UNIX command-line application – the same syntax for options and arguments, and the same usage text format. decline’s behaviour is based on the POSIX utility conventions, GNU’s extensions, and reference to existing, popular command line tooling.

  • Performance: While raw performance is not the most important design concern, it’s important that the performance is predictable. In particular, parsing should take time and space linear in the size of the input.
